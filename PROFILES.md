# Несколько профилей для разных сетей

В TG WS Proxy профиль — это «вариант пути до Telegram». Под разными
сетями один и тот же путь работает по-разному: домашний провайдер
обычно пускает напрямую, а у мобильного оператора часто нужен обход
через Cloudflare. Автопереключение между профилями делает это за вас.

После онбординга у вас уже есть один профиль «Основной» с
включённым Cloudflare-обходом — он подходит как «безопасный default»,
который работает на большинстве сетей. Превратим его в мобильный
профиль и добавим к нему второй — для домашнего Wi-Fi без обходов.

## Сценарий

- Мобильный оператор — нужен Cloudflare.
- Дом — Wi-Fi, прямое подключение работает быстрее.

## Шаги

> Меню профиля (Настроить / Закрепить / Переименовать / Дублировать /
> Удалить) открывается как через три точки на чипе, так и долгим
> тапом по самому чипу — что удобнее.

1. **Переименуйте «Основной» в `Мобильный`**
   - Меню профиля → **Переименовать**.
   - В появившемся диалоге введите `Мобильный` → подтвердите.

2. **Настройте профиль `Мобильный`**
   - Меню профиля → **Настроить**.
   - Cloudflare-обход: оставьте **включённым** (был включён после
     онбординга).
   - Автопереключение: **Мобильная сеть**.
   - Сохранить.

3. **Создайте и настройте второй профиль**
   - Главный экран → **+** в конце списка профилей (или меню профиля
     → **Дублировать**, если хочется начать с готового набора
     настроек).
   - В появившемся диалоге введите `Дом` → подтвердите. Откроется
     экран редактирования.
   - Cloudflare-обход: **выключите**.
   - Автопереключение: **Wi-Fi**.
   - Сохранить.

Готово. При переключении телефона между Wi-Fi и мобильной сетью
приложение само выберет нужный профиль.

## Как работает автопереключение

Каждый тип сети (Wi-Fi или Cellular) может быть привязан только к
одному профилю — в карточке настройки профиля секция «Автопереключение»
не даст выбрать тип, который уже занят другим профилем.

При смене активной сети приложение находит профиль, привязанный к
её типу, и делает его активным. Если ни один профиль не привязан
к текущему типу сети, активный профиль не меняется.

Чтобы временно отключить автопереключение целиком — закрепите
любой профиль через пункт **Закрепить** в меню. До ручного открепления
активный профиль не сменится, какая бы сеть ни появилась.

URL прокси, который Telegram запомнил при онбординге, **общий на
все профили** — добавлять новые подключения в Telegram'е после
создания нового профиля не нужно. Сменится только то, как
приложение обрабатывает трафик после получения.

## Частые ошибки

- **«После смены сети Telegram молчит ~10 секунд»** — это нормально:
  идёт graceful-rebind на новый профиль. Соединение восстановится.
- **«Хочу один профиль, чтобы прокси никогда не переключался
  автоматом»** — закрепите его иконкой замка. Автопереключение
  перестанет менять активный профиль до снятия пина.
